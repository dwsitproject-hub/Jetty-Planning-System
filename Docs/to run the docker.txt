What to change (high impact)
1) Stop bind-mounting uploads from Windows host
Use container-local or named volume for uploads.

In Backend/docker-compose.yml, keep:

UPLOAD_DIR: /tmp/jps-uploads (or better named volume path)
remove ./uploads:/app/uploads
This avoids the Windows mount path that hangs during writes.

2) Move project into WSL2 filesystem (very important)
Do not run repo from D:\... for Docker bind mounts if possible.
Use path like:

/home/<user>/jetty-planning-system (inside WSL ext4)
Then run docker compose from WSL terminal.
This alone usually fixes 80% of Windows Docker file I/O hangs.

3) Avoid --build on every restart
Use daily:

docker compose up -d Only use build when Dockerfile/dependencies changed:
docker compose up -d --build
4) Reduce file-watch churn
Your API runs node --watch; with Windows mounts this can trigger frequent restarts. If still unstable, use plain node src/index.js inside container during troubleshooting.

5) Exclude Docker/workspace from antivirus scanning
Windows Defender / Controlled Folder Access can cause exactly this behavior on mounted paths.

Recovery sequence when it gets stuck
When Docker is jammed (cannot stop container), do this:

Quit Docker Desktop fully
Run wsl --shutdown
Start Docker Desktop again
Wait healthy
Recreate cleanly:
docker compose down --remove-orphans
docker compose up -d
